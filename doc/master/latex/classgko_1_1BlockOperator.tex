\doxysubsection{Detailed Description}
A \mbox{\hyperlink{classgko_1_1BlockOperator}{Block\+Operator}} represents a linear operator that is partitioned into multiple blocks. 

For example, a \mbox{\hyperlink{classgko_1_1BlockOperator}{Block\+Operator}} can be used to define the operator\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{| A B |}
\DoxyCodeLine{| C D |}
\end{DoxyCode}
 where {\ttfamily A, B, C, D} itself are matrices of compatible size. This can be created with\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{ \{c++\}}
\DoxyCodeLine{std::shared\_ptr<LinOp> A = ...;}
\DoxyCodeLine{std::shared\_ptr<LinOp> B = ...;}
\DoxyCodeLine{std::shared\_ptr<LinOp> C = ...;}
\DoxyCodeLine{std::shared\_ptr<LinOp> D = ...;}
\DoxyCodeLine{\textcolor{keyword}{auto} bop = \mbox{\hyperlink{classgko_1_1BlockOperator_a8a8952f69331001fc37ae48ae97744b9}{BlockOperator::create}}(exec, \{\{A, B\}, \{C, D\}\});}
\end{DoxyCode}
 The requirements on the individual blocks passed to the create method are\+:
\begin{DoxyItemize}
\item In each block-\/row, all blocks have the same number of rows
\item In each block-\/column, all blocks have the same number of columns
\item Each block-\/row must have the same number of blocks It is possible to set blocks to zero, by passing in a nullptr. But every block-\/row and block-\/column must contain at least one non-\/nullptr block.
\end{DoxyItemize}

The constructor will store all passed in blocks on the same executor as the \mbox{\hyperlink{classgko_1_1BlockOperator}{Block\+Operator}}, which will requires copying any block that is associated with a different executor.
