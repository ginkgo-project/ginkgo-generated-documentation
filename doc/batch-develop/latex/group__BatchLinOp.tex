\doxysubsection{Detailed Description}
\hypertarget{group__BatchLinOp_batch_linop_concept}{}\doxysubsection{Batched Linear operator as a concept}\label{group__BatchLinOp_batch_linop_concept}
A batch linear operator (\mbox{\hyperlink{classgko_1_1BatchLinOp}{Batch\+Lin\+Op}}) forms the base class for all batched linear algebra objects. In general, it follows the same structure as the \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} class, but has some crucial differences which make it not strictly representable through or with the \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} class.

A batched operator is defined as a set of independent linear operators which have no communication/information exchange between them. Therefore, any collective operations between the batches is not possible and not implemented. This allows for each batch to be computed and operated on in an embarrasingly parallel fashion.

Similar to the \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} class, the \mbox{\hyperlink{classgko_1_1BatchLinOp}{Batch\+Lin\+Op}} also implements \mbox{\hyperlink{classgko_1_1BatchLinOp_aeaf630de43cfd9e623e69299def5ee69}{Batch\+Lin\+Op\+::apply()}} methods which call the internal apply\+\_\+impl() methods which the concrete \mbox{\hyperlink{classgko_1_1BatchLinOp}{Batch\+Lin\+Op}}\textquotesingle{}s have to implement.

A key difference between the \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} and the \mbox{\hyperlink{classgko_1_1BatchLinOp}{Batch\+Lin\+Op}} classes is the storing of dimensions. \mbox{\hyperlink{classgko_1_1BatchLinOp}{Batch\+Lin\+Op}} allows for storing non-\/equal objects in the batches and hence stores a \mbox{\hyperlink{structgko_1_1batch__dim}{batch\+\_\+dim}} object instead of a dim object. The \mbox{\hyperlink{structgko_1_1batch__dim}{batch\+\_\+dim}} object is optimized to store less amount of data when storing uniform batches.

All size validation functions again verify first that the number of batches are conformant and that the dimensions in the corresponding batches themselves are also valid/conformant. Here too, optimizations for uniform batches have been added.
