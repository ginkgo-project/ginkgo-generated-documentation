\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Value\+Type, typename Index\+Type$>$\newline
class gko\+::device\+\_\+matrix\+\_\+data$<$ Value\+Type, Index\+Type $>$}

This type is a device-\/side equivalent to \mbox{\hyperlink{structgko_1_1matrix__data}{matrix\+\_\+data}}. 

It stores the data necessary to initialize any matrix format in Ginkgo in individual value, column and row index arrays together with associated matrix dimensions. \mbox{\hyperlink{structgko_1_1matrix__data}{matrix\+\_\+data}} uses Array-\/of-\/\+Structs storage (AoS), while \mbox{\hyperlink{classgko_1_1device__matrix__data}{device\+\_\+matrix\+\_\+data}} uses Struct-\/of-\/\+Arrays (SoA).

\begin{DoxyNote}{Note}
To be used with a Ginkgo matrix type, the entry array must be sorted in row-\/major order, i.\+e. by row index, then by column index within rows. This can be achieved by calling the sort\+\_\+row\+\_\+major function. 

The data must not contain any duplicate (row, column) pairs.
\end{DoxyNote}
