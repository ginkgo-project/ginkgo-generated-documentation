The {\ttfamily system\+\_\+matrix}, which will be given to this factory, must be sorted (first by row, then by column) in order for the algorithm to work. 

If it is known that the matrix will be sorted, this parameter can be set to {\ttfamily true} to skip the sorting (therefore, shortening the runtime). However, if it is unknown or if the matrix is known to be not sorted, it must remain {\ttfamily false}, otherwise, the algorithm may produce incorrect results or crash. \mbox{\Hypertarget{structgko_1_1experimental_1_1factorization_1_1Cholesky_1_1parameters__type_a4404b9835073e6377135fbe170315a41}\label{structgko_1_1experimental_1_1factorization_1_1Cholesky_1_1parameters__type_a4404b9835073e6377135fbe170315a41}}

The combined sparsity pattern L + L$^\wedge$H of the factors L and L$^\wedge$H. 

It can be used to avoid the potentially costly symbolic factorization of the system matrix if its symbolic factorization is already known. If it is set to nullptr, the symbolic factorization will be computed.
