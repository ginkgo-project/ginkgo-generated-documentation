\subsubsection*{template$<$typename Abstract\+Product\+Type, typename Components\+Type$>$\newline
class gko\+::\+Abstract\+Factory$<$ Abstract\+Product\+Type, Components\+Type $>$}

The \mbox{\hyperlink{classgko_1_1AbstractFactory}{Abstract\+Factory}} is a generic interface template that enables easy implementation of the abstract factory design pattern. 

The interface provides the \mbox{\hyperlink{classgko_1_1AbstractFactory_a8bb9a83878530553e63d2f92e7b3e5c2}{Abstract\+Factory\+::generate()}} method that can produce products of type {\ttfamily Abstract\+Product\+Type} using an object of {\ttfamily Components\+Type} (which can be constructed on the fly from parameters to its constructors). The \mbox{\hyperlink{classgko_1_1AbstractFactory_a8bb9a83878530553e63d2f92e7b3e5c2}{generate()}} method is not declared as virtual, as this allows subclasses to hide the method with a variant that preserves the compile-\/time type of the objects. Instead, implementers should override the generate\+\_\+impl() method, which is declared virtual.

Implementers of concrete factories should consider using the \mbox{\hyperlink{classgko_1_1EnableDefaultFactory}{Enable\+Default\+Factory}} mixin to obtain default implementations of utility methods of \mbox{\hyperlink{classgko_1_1PolymorphicObject}{Polymorphic\+Object}} and \mbox{\hyperlink{classgko_1_1AbstractFactory}{Abstract\+Factory}}.
