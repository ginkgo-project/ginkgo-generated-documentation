template<typename ValueType = default_precision>
bool gko::solver::BatchIdr< ValueType >::parameters_type::complex_subspace { false }
mutable

If set to true, IDR is supposed to use a complex subspace S also for real problems, allowing for faster convergence and better results by acknowledging the influence of complex eigenvectors.

The default is false. Currently, the option of having complex subspace for real matrices is not supported.

Referenced by gko::solver::BatchIdr< ValueType >::get_complex_subspace(), and gko::solver::BatchIdr< ValueType >::set_complex_subspace().

◆ deterministic

template<typename ValueType = default_precision>
bool gko::solver::BatchIdr< ValueType >::parameters_type::deterministic { false }
mutable

If set to true, the vectors spanning the subspace S are chosen deterministically.

This is mostly needed for testing purposes.

Note: If 'deterministic' is set to true, the subspace vectors are generated in serial on the CPU, which can be very slow.

The default behaviour is to choose the subspace vectors randomly.

Referenced by gko::solver::BatchIdr< ValueType >::get_deterministic(), and gko::solver::BatchIdr< ValueType >::set_deterministic().
