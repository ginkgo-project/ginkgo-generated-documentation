\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Value\+Type = default\+\_\+precision, typename Local\+Index\+Type = int32, typename Global\+Index\+Type = int64$>$\newline
class gko\+::experimental\+::distributed\+::\+Matrix$<$ Value\+Type, Local\+Index\+Type, Global\+Index\+Type $>$}

The \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Matrix}{Matrix}} class defines a (M\+P\+I-\/)distributed matrix. 

The matrix is stored in a row-\/wise distributed format. Each process owns a specific set of rows, where the assignment of rows is defined by a row \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Partition}{Partition}}. The following depicts the distribution of global rows according to their assigned part-\/id (which will usually be the owning process id)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Part-\/Id  Global Rows                   Part-\/Id  Local Rows}
\DoxyCodeLine{0        | .. 1  2  .. .. .. |         0        | .. 1  2  .. .. .. |}
\DoxyCodeLine{1        | 3  4  .. .. .. .. |                  | 13 .. .. .. 14 .. |}
\DoxyCodeLine{2        | .. 5  6  ..  7 .. |  -\/-\/-\/-\/>  1        | 3  4  .. .. .. .. |}
\DoxyCodeLine{2        | .. .. .. 8  ..  9 |  -\/-\/-\/-\/>           | .. .. .. 10 11 12 |}
\DoxyCodeLine{1        | .. .. .. 10 11 12 |         2        | .. 5  6  ..  7 .. |}
\DoxyCodeLine{0        | 13 .. .. .. 14 .. |                  | .. .. .. 8  ..  9 |}
\end{DoxyCode}
 The local rows are further split into two matrices on each process. One matrix, called {\ttfamily local}, contains only entries from columns that are also owned by the process, while the other one, called {\ttfamily non\+\_\+local}, contains entries from columns that are not owned by the process. The non-\/local matrix is stored in a compressed format, where empty columns are discarded and the remaining columns are renumbered. This splitting is depicted in the following\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Part-\/Id  Global                            Local      Non-\/Local}
\DoxyCodeLine{0        | .. 1  ! 2  .. ! .. .. |         | .. 1  |  | 2  |}
\DoxyCodeLine{0        | 3  4  ! .. .. ! .. .. |         | 3  4  |  | .. |}
\DoxyCodeLine{         |-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/|}
\DoxyCodeLine{1        | .. 5  ! 6  .. ! 7  .. |  -\/-\/-\/-\/>  | 6  .. |  | 5  7  .. |}
\DoxyCodeLine{1        | .. .. ! .. 8  ! ..  9 |  -\/-\/-\/-\/>  | 8  .. |  | .. .. 9  |}
\DoxyCodeLine{         |-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/|}
\DoxyCodeLine{2        | .. .. ! .. 10 ! 11 12 |         | 11 12 |  | .. 10 |}
\DoxyCodeLine{2        | 13 .. ! .. .. ! 14 .. |         | 14 .. |  | 13 .. |}
\end{DoxyCode}
 This uses the same ownership of the columns as for the rows. Additionally, the ownership of the columns may be explicitly defined with an second column partition. If that is not provided, the same row partition will be used for the columns. Using a column partition also allows to create non-\/square matrices, like the one below\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Part-\/Id  Global                  Local      Non-\/Local}
\DoxyCodeLine{P\_R/P\_C    2  2  0  1}
\DoxyCodeLine{0        | .. 1  2  .. |         | 2  |     | 1  .. |}
\DoxyCodeLine{0        | 3  4  .. .. |         | .. |     | 3  4  |}
\DoxyCodeLine{         |-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/|}
\DoxyCodeLine{1        | .. 5  6  .. |  -\/-\/-\/-\/>  | .. |     | 6  5  |}
\DoxyCodeLine{1        | .. .. .. 8  |  -\/-\/-\/-\/>  | 8  |     | .. .. |}
\DoxyCodeLine{         |-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/|}
\DoxyCodeLine{2        | .. .. .. 10 |         | .. .. |  | 10 |}
\DoxyCodeLine{2        | 13 .. .. .. |         | 13 .. |  | .. |}
\end{DoxyCode}
 Here {\ttfamily P\+\_\+R} denotes the row partition and {\ttfamily P\+\_\+C} denotes the column partition.

The \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Matrix}{Matrix}} should be filled using the read\+\_\+distributed method, e.\+g. 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto} part = \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Partition_a196576140065691b5726ce1cb286302c}{Partition<...>::build\_from\_mapping}}(...);}

\end{DoxyCode}
 or if different partitions for the rows and columns are used\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto} row\_part = \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Partition_a196576140065691b5726ce1cb286302c}{Partition<...>::build\_from\_mapping}}(...);}

 This will set the dimensions of the global and local matrices automatically by deducing the sizes from the partitions.

By default the \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Matrix}{Matrix}} type uses Csr for both stored matrices. It is possible to explicitly change the datatype for the stored matrices, with the constraint that the new type should implement the \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} and \mbox{\hyperlink{classgko_1_1ReadableFromMatrixData}{Readable\+From\+Matrix\+Data}} interface.
