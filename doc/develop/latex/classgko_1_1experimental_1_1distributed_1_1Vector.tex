\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Value\+Type = double$>$\newline
class gko\+::experimental\+::distributed\+::\+Vector$<$ Value\+Type $>$}

\mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Vector}{Vector}} is a format which explicitly stores (multiple) distributed column vectors in a dense storage format. 

The (multi-\/)vector is distributed by row, which is described by a \begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Partition}{Partition}}. The local vectors are stored using the 

Dense format. The vector should be filled using the \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Vector_aa9ed9afbbc309bb30ad65e94e9c965ec}{read\+\_\+distributed}} method, e.\+g. 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto} part = \mbox{\hyperlink{classgko_1_1experimental_1_1distributed_1_1Partition_a196576140065691b5726ce1cb286302c}{Partition<...>::build\_from\_mapping}}(...);}
\DoxyCodeLine{\textcolor{keyword}{auto} vector = Vector<...>::create(exec, comm);}
\DoxyCodeLine{vector-\/>read\_distributed(matrix\_data, part);}
\end{DoxyCode}
 Using this approach the size of the global vectors, \mbox{\hyperlink{namespacegko_a73ce7e87aec389b5210630bb617b4baa}{as}} well \mbox{\hyperlink{namespacegko_a73ce7e87aec389b5210630bb617b4baa}{as}} the size of the local vectors, will be automatically inferred.
