\doxysubsection{Detailed Description}
This class describes the value and index types to be used when building a Ginkgo type from a configuration file. 

A \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor}{type\+\_\+descriptor}} is passed in order to define the parse function defines which template parameters, in terms of value\+\_\+type and/or index\+\_\+type, the created object will have. For example, a CG solver created like this\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto} cg = parse(config, context, \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor_a174c5d07919038e0f5858ca25d55002d}{type\_descriptor}}(\textcolor{stringliteral}{"float64"}, \textcolor{stringliteral}{"int32"}));}
\end{DoxyCode}
 will have the value type {\ttfamily float64} and the index type {\ttfamily int32}. Any Ginkgo object that does not require one of these types will just ignore it. In value\+\_\+type, one additional value {\ttfamily void} can be used to specify that no default type is provided. In this case, the configuration has to provide the necessary template types.

If the configuration specifies one field (only allow value\+\_\+type now)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{value\_type: \textcolor{stringliteral}{"some\_value\_type"}}
\end{DoxyCode}
 this type will take precedence over the \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor}{type\+\_\+descriptor}}.
