The {\ttfamily system\+\_\+matrix}, which will be given to this factory, must be sorted (first by row, then by column) in order for the algorithm to work. 

If it is known that the matrix will be sorted, this parameter can be set to {\ttfamily true} to skip the sorting (therefore, shortening the runtime). However, if it is unknown or if the matrix is known to be not sorted, it must remain {\ttfamily false}, otherwise, the algorithm may produce incorrect results or crash. \mbox{\Hypertarget{structgko_1_1experimental_1_1factorization_1_1Lu_1_1parameters__type_a5b9ba2e6568698533c54dde0def07a14}\label{structgko_1_1experimental_1_1factorization_1_1Lu_1_1parameters__type_a5b9ba2e6568698533c54dde0def07a14}}

The combined sparsity pattern L + U of the factors L and U. 

It can be used to avoid the potentially costly symbolic factorization of the system matrix if its symbolic factorization is already known. If it is set to nullptr, the symbolic factorization will be computed. \begin{DoxyNote}{Note}
Currently, the symbolic factorization needs to be provided if the system matrix does not have a symmetric sparsity pattern.
