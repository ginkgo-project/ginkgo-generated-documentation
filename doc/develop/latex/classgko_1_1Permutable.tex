\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Index\+Type$>$\newline
class gko\+::\+Permutable$<$ Index\+Type $>$}

Linear operators which support permutation should implement the \mbox{\hyperlink{classgko_1_1Permutable}{Permutable}} interface. 

It provides functions to permute the rows and columns of a \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}}, independently or symmetrically, and with a regular or inverted permutation.

After a regular row permutation with permutation array {\ttfamily perm} the row {\ttfamily i} in the output \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} contains the row {\ttfamily perm\mbox{[}i\mbox{]}} from the input \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}}. After an inverse row permutation, the row {\ttfamily perm\mbox{[}i\mbox{]}} in the output \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}} contains the row {\ttfamily i} from the input \mbox{\hyperlink{classgko_1_1LinOp}{Lin\+Op}}. Equivalently, after a column permutation, the output stores in column {\ttfamily i} the column {\ttfamily perm\mbox{[}i\mbox{]}} from the input, and an inverse column permutation stores in column {\ttfamily perm\mbox{[}i\mbox{]}} the column {\ttfamily i} from the input. A symmetric permutation is functionally equivalent to calling {\ttfamily as$<$\mbox{\hyperlink{classgko_1_1Permutable}{Permutable}}$>$(A-\/$>$row\+\_\+permute(perm))-\/$>$column\+\_\+permute(perm)}, but the implementation can provide better performance due to kernel fusion.\hypertarget{classgko_1_1Permutable_autotoc_md77}{}\doxysubsubsection{Example\+: Permuting a Csr matrix\+:}\label{classgko_1_1Permutable_autotoc_md77}
