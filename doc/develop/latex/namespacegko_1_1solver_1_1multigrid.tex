cycle defines which kind of multigrid cycle can be used. 

It contains V, W, F, and K (K\+F\+C\+G/\+K\+G\+CR) cycle.
\begin{DoxyItemize}
\item V, W cycle uses the algorithm according to Briggs, Henson, and Mc\+Cormick\+: A multigrid tutorial 2nd Edition.
\item F cycle uses the algorithm according to Trottenberg, Oosterlee, and Schuller\+: \mbox{\hyperlink{classgko_1_1solver_1_1Multigrid}{Multigrid}} 1st Edition. F cycle first uses the recursive call but second uses the V-\/cycle call such that F-\/cycle is between V and W cycle.
\item K(K\+F\+C\+G/\+K\+G\+CR) cycle uses the algorithm with up to 2 steps F\+C\+G/\+G\+CR from Yvan\+: An aggregation-\/based algebraic multigrid method 
\end{DoxyItemize}\mbox{\Hypertarget{namespacegko_1_1solver_1_1multigrid_a24ab73573aae583623848c9bfaf0fddf}\label{namespacegko_1_1solver_1_1multigrid_a24ab73573aae583623848c9bfaf0fddf}}

{\footnotesize\ttfamily enum \mbox{\hyperlink{namespacegko_1_1solver_1_1multigrid_a24ab73573aae583623848c9bfaf0fddf}{gko\+::solver\+::multigrid\+::mid\+\_\+smooth\+\_\+type}}\hspace{0.3cm}{\ttfamily [strong]}}



mid\+\_\+smooth\+\_\+type gives the options to handle the middle smoother behavior between the two cycles in the same level. 

It only affects the behavior when there\textquotesingle{}s no operation between the post smoother of previous cycle and the pre smoother of next cycle. Thus, it only affects W cycle and F cycle.
\begin{DoxyItemize}
\item both\+: gives the same behavior as the original algorithm, which use posts smoother from previous cycle and pre smoother from next cycle.
\item post\+\_\+smoother\+: only uses the post smoother of previous cycle in the mid smoother
\item pre\+\_\+smoother\+: only uses the pre smoother of next cycle in the mid smoother
\item standalone\+: uses the defined smoother in the mid smoother 
\end{DoxyItemize}
