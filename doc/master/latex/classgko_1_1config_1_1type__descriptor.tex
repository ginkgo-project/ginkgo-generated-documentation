\doxysubsection{Detailed Description}
This class describes the value and index types to be used when building a Ginkgo type from a configuration file. 

A \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor}{type\+\_\+descriptor}} is passed in order to define the parse function defines which template parameters, in terms of value\+\_\+type and/or index\+\_\+type, the created object will have. For example, a CG solver created like this\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto} cg = parse(config, context, \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor_a2edc43c2f174dce3e933951ef028af49}{type\_descriptor}}(\textcolor{stringliteral}{"float64"}, \textcolor{stringliteral}{"int32"}));}
\end{DoxyCode}
 will have the value type {\ttfamily float64} and the index type {\ttfamily int32}. Any Ginkgo object that does not require one of these types will just ignore it. The value {\ttfamily void} can be used to specify that no default type is provided. In this case, the configuration has to provide the necessary template types.

If the configuration specifies one of the fields (or both)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{value\_type: \textcolor{stringliteral}{"some\_value\_type"}}
\DoxyCodeLine{index\_type: \textcolor{stringliteral}{"some\_index\_type"}}
\end{DoxyCode}
 these types will take precedence over the \mbox{\hyperlink{classgko_1_1config_1_1type__descriptor}{type\+\_\+descriptor}}.
